# 디자이너도 웹앱을 만들 수 있어요

코딩 몰라도 됩니다. 시안 이미지와 기획 내용을 주면, **AI가 직접 코드를 짜고 배포까지** 해 드립니다.
디자이너는 채팅만 하면 됩니다.

---

## 어떻게 동작하나요?

이 폴더는 AI(Claude)가 다음 10단계를 직접 실행하는 작업 공간입니다.

```
1. GitHub 레포 생성 + Vercel 배포 준비
2. 프로젝트 정보 수집
3. 시안 이미지 분석
4. 요구사항(PRD) 작성
5. 필요한 정보 수집 (API 키 등)
6. DB 설계
7. Supabase(DB) 세팅
8. 코드 직접 구현 + PR 생성
9. 피드백 반영 · 수정
10. 배포(Merge)
```

매 단계마다 AI가 "지금 뭘 해야 하는지" 채팅으로 안내합니다.
막히면 화면 캡처를 보내 주세요. AI가 바로 다음 단계를 알려줍니다.

---

## 시작 전에 필요한 것

딱 3개 서비스 가입이 필요합니다. 전부 무료로 시작할 수 있어요.

| 서비스 | 하는 일 | 가입 링크 |
|--------|---------|-----------|
| **GitHub** | 코드 저장 공간 | [github.com/join](https://github.com/join) |
| **Vercel** | 웹 앱 배포 (GitHub로 로그인 가능) | [vercel.com/signup](https://vercel.com/signup) |
| **Supabase** | 데이터베이스 (7번 단계에서 씁니다) | [supabase.com/dashboard](https://supabase.com/dashboard) |

> 가입만 해 두시면 됩니다. 설정은 AI가 단계별로 안내합니다.

---

## 시작하는 방법

이 폴더를 Claude Code에서 열고, 채팅에 딱 한 마디만 입력하세요.

```
시작해줘
```

그러면 AI가 현재 상태를 확인하고, 다음 할 일을 바로 알려 줍니다.

---

## 10단계 흐름 — 디자이너가 할 일 vs AI가 할 일

### 1단계 — GitHub 레포 + Vercel 배포

| 역할 | 할 일 |
|------|-------|
| 디자이너 | 프로젝트 이름(영문 소문자 + 하이픈) 알려주기. 예: `my-portfolio` |
| AI | GitHub 레포 생성, Next.js 보일러플레이트 설치, Vercel 배포 직접 실행 |

> `gh`, `vercel` CLI가 없으면 AI가 직접 설치합니다. Mac은 Homebrew도 함께 설치됩니다.
> GitHub/Vercel 로그인은 브라우저 인증이 한 번 필요합니다 — AI가 터미널 여는 법부터 안내합니다.
> `gh auth login` 실행 시 반드시 **"Login with a web browser"** 를 선택하세요. 토큰 직접 입력 방식은 사용하지 마세요.

---

### 2단계 — 프로젝트 정보 수집

| 역할 | 할 일 |
|------|-------|
| 디자이너 | 프로젝트명, GitHub URL, Vercel URL 채팅으로 알려주기 |
| AI | `project/info.md`에 저장 |

> GitHub URL은 레포 주소창, Vercel URL은 대시보드 → 프로젝트 → Visit 링크에서 확인할 수 있어요.

---

### 3단계 — 시안 이미지 분석

| 역할 | 할 일 |
|------|-------|
| 디자이너 | Figma 또는 Stitch 시안 이미지를 채팅에 첨부 |
| AI | 이미지를 직접 읽고 화면 스펙 추출, `project/screens/`에 저장 |

> 이미지만 있어도 충분합니다. "이 화면은 메인 화면이에요"처럼 짧은 설명을 붙여 주시면 더 정확합니다.
> 시안이 없으면 [Stitch](https://stitch.withgoogle.com/)를 추천해요.

---

### 4단계 — 요구사항(PRD) 작성

| 역할 | 할 일 |
|------|-------|
| 디자이너 | 시안 + 기획 의도 전달 → PRD 검토 후 OK 또는 수정 요청 |
| AI | 기획자 + 개발자 관점으로 PRD 작성, `project/PRD.md`에 저장 |

> PRD = 앱이 어떻게 동작해야 하는지 정리한 문서입니다.
> 내용이 맞는지 확인하고 "OK" 또는 "이 부분 바꿔줘"라고만 하면 됩니다.

---

### 5단계 — 필요한 정보 수집

| 역할 | 할 일 |
|------|-------|
| 디자이너 | AI가 채팅으로 물어보는 것에 답변 (API 키, 서비스명 등) |
| AI | 답변을 `project/required-info.md`에 저장 |

> 외부 데이터(날씨, 주식 등)가 필요한 경우 API 키를 물어봅니다.
> 모르는 게 있으면 "어떻게 발급해요?"라고 하면 AI가 바로 안내합니다.

---

### 6단계 — DB 설계

| 역할 | 할 일 |
|------|-------|
| 디자이너 | 없음 |
| AI | 시안·PRD 기반으로 DB 테이블 설계 및 SQL 작성, `project/db-design.md`에 저장 |

> 디자이너는 기다리시면 됩니다.

---

### 7단계 — Supabase 세팅

| 역할 | 할 일 |
|------|-------|
| 디자이너 | Supabase에서 프로젝트 생성 → SQL 실행 → URL·키 채팅으로 전달 |
| AI | URL·키를 받아 `project/info.md`에 반영, Vercel 환경변수 자동 설정 |

**순서**:
1. [Supabase 대시보드](https://supabase.com/dashboard)에서 **New project** 생성
2. 왼쪽 메뉴 **SQL Editor** → `project/db-design.md` 파일 맨 아래 SQL 전체 복사 → **Run**
3. **Project Settings(톱니바퀴) → API** 탭에서 **Project URL**과 **anon public** 키를 채팅으로 보내 주세요

> Supabase MCP가 연결된 환경이면 "7번 자동으로 해줘"라고 하면 AI가 직접 실행합니다.
> 막히면 화면 캡처를 채팅에 보내 주세요.

---

### 8단계 — 코드 직접 구현

| 역할 | 할 일 |
|------|-------|
| 디자이너 | 기다리기. 완료 후 Preview URL 확인 |
| AI | GitHub 클론 → Next.js 코드 작성 → PR 생성 → Preview URL 전달 |

> PR(Pull Request) = AI가 작성한 코드를 검토용으로 올린 것입니다.
> Preview URL = 실제 배포 전 미리 볼 수 있는 링크입니다. 보통 1~2분 내에 생성돼요.

---

### 9단계 — 피드백 · 수정

| 역할 | 할 일 |
|------|-------|
| 디자이너 | Preview URL 확인 후 수정할 것 채팅으로 요청 |
| AI | 코드 직접 수정 → push → Preview URL 자동 업데이트 |

수정 요청은 이 포맷이 가장 명확해요:
> `[페이지] + [무엇을] + [어떻게 바꿔]`
> 예: `메인 화면 카드 간격 더 넓혀줘` / `버튼 색을 파란색으로 바꿔줘`

만족할 때까지 반복할 수 있습니다.

---

### 10단계 — 배포(Merge)

| 역할 | 할 일 |
|------|-------|
| 디자이너 | "10번 Merge해줘" 한 마디 |
| AI | `gh pr merge`로 자동 반영 → Vercel이 자동 배포 |

> Merge 후 1~2분 내로 실제 URL에서 확인할 수 있어요.
> 이후 추가 수정이 필요하면 9번부터 다시 시작하면 됩니다.

## 자주 묻는 것

**Q. 코딩을 전혀 몰라도 되나요?**
네, 전혀 몰라도 됩니다. 채팅으로 대화하는 것 외에 코드를 볼 일이 없어요.

**Q. 막히면 어떻게 하나요?**
화면 캡처를 채팅에 붙여 주세요. AI가 바로 다음 단계를 안내합니다.
또는 "지금 어디야?" / "다음 뭐 해야 해?"라고 물어보면 현재 상태를 알려 줍니다.

**Q. 중간에 그만뒀다가 다시 시작하면?**
`project/state.md`에 진행 상태가 저장돼 있어서 이어서 진행됩니다.
채팅에 "시작해줘"라고만 하면 AI가 어디까지 됐는지 확인하고 이어 갑니다.

**Q. 시안 없이 시작할 수 있나요?**
[Stitch](https://stitch.withgoogle.com/)에서 간단한 화면 시안을 만들 수 있어요.
아이디어만 있어도 AI가 화면 구성을 도와드릴 수 있습니다.
