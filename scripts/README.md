# KOSPI Top 10 크롤 로직 검증

**결론: 현재 크롤 코드는 정상 동작합니다.** (10건 추출 확인됨)  
empty view가 뜬다면 **API 경로 호출** 또는 **프론트에서 데이터 바인딩** 쪽을 확인하면 됩니다.

## 이 폴더에서 확인하는 방법

```bash
cd scripts
npm install
npm run test:crawl
```

- **10개 나오면** → 크롤 로직 OK. 줄스/앱 쪽에서 API 라우트가 이 함수를 호출하는지, 프론트가 응답을 그리는지 확인.
- **0개 나오면** → 네이버 페이지 구조가 바뀌었을 수 있음. `npm run debug:html` 로 테이블/셀렉터 개수·HTML 스니펫 확인.

## 실제 앱(줄스 레포)에서 확인하려면

1. **API 라우트**  
   해당 크롤 함수를 호출하는 API(예: `/api/kospi-top10`)가 있는지, 브라우저나 `curl`로 직접 호출해서 JSON이 오는지 확인.

2. **프론트**  
   그 API를 호출한 뒤 `stocks.length === 0`일 때만 empty view가 나오는지, 응답을 state에 넣고 리스트를 렌더링하는지 확인.

3. **환경**  
   Vercel 등 서버리스에서는 첫 요청이 느리거나 타임아웃될 수 있음. 로컬에서 같은 API를 호출해 보면 구분 가능.
