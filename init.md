# init.md — 프로젝트 세팅

이 폴더에서 **디자인 → 웹앱** 과정을 시작할 때, **가장 먼저** 실행하는 파일입니다.

**처음 이 폴더에 들어왔다면** 채팅에 **"시작해줘"** 또는 **"flow-guide 읽고 다음 진행해줘"**라고만 입력하세요. AI가 init 필요 여부부터 현재 단계까지 안내합니다. (파일은 나중에 보고, 채팅에서 한 마디만 하면 됩니다.)

---

## 실행 방법 (AI가 "init 실행해줘"라고 안내했을 때)

채팅에서 아래처럼 입력하세요.

```
init.md 실행해줘
```

AI가 이 파일을 읽고, **아래 "생성할 구조"**대로 `project/` 폴더와 필요한 파일을 모두 생성합니다.  
한 번 실행하면 세팅이 끝나고, 이후에는 `flow-guide.md`와 `project/`만 보면서 진행하면 됩니다.

---

## 이 플로우에 필요한 서비스

디자이너가 막힘 없이 따라가려면 **GitHub, Vercel, Jules, Supabase** 네 서비스 **가입**이 필요합니다. 프로젝트/레포는 1·7·8단계에서 만들면 됩니다. 처음이시면 "이 네 서비스 가입만 해 두시면, 1번부터 순서대로 안내해 드릴게요"라고 안내해 주세요.

| 서비스 | 용도 |
|--------|------|
| GitHub | 코드 저장. 1번에서 Vercel 연동 시 사용 |
| Vercel | 웹 배포. 1번에서 템플릿 배포 |
| Jules | 구현 AI. 8·9번에서 프롬프트 전달·수정 |
| Supabase | DB. 7번에서 프로젝트 생성·SQL 실행 |

---

## 생성할 구조 (실행 시 만들어지는 것)

| 경로 | 용도 |
|------|------|
| `project/` | 이번 프로젝트의 모든 입력·산출물이 들어가는 폴더 (flow-guide는 덮어쓰지 않음) |
| `project/state.md` | 현재 단계(stage), 다음 액션(next). AI가 여기를 보고 다음 할 일을 안내함 |
| `project/info.md` | 프로젝트명, GitHub URL, Vercel URL, Supabase URL 등 (1·2번 단계에서 채움) |
| `project/screens/` | 시안(Stitch/Figma) 이미지를 저장하는 폴더. 채팅으로 받은 이미지는 여기 저장 |
| `project/screens/README.md` | 화면별 한 줄 설명을 적는 곳 (3번 단계) |
| `project/PRD.md` | A–Z 할 일을 정리한 Plan/PRD. 4번 Plan 생성 (및 검토) 단계에서 AI가 시안 기반으로 생성 |
| `project/required-info.md` | 프로젝트에 필요한 추가 정보(키, 서비스명·요금 등). 5번 필요한 정보 수집에서 채움 |
| `project/db-design.md` | DB 설계 결과(테이블, DDL, 목데이터). 6번 단계에서 채움 |
| `project/jules-prompt.md` | Jules에 붙여넣을 프롬프트. init 시 최소 템플릿으로 생성하고, 8단계에서 AI가 PRD와 기본 포맷을 머징해 채움 |
| `project/README.md` | project/ 폴더 설명 (어떤 파일에 뭘 채우는지) |

**jules-prompt.md 초기 내용** (생성 시 넣을 최소 템플릿):
- 제목: `# Jules용 복붙 프롬프트 (8·9단계)`
- 한 줄: "8단계에서 AI가 PRD와 기본 포맷을 머징해 이 파일을 채웁니다. 그때 나온 블록을 Jules에 붙여넣으세요."
- "수정 요청 시 (9단계)" 섹션: "수정하고 싶은 게 있으면 AI에게 알려주세요. 논의 후 AI가 아래에 복붙용 블록을 추가합니다. AI가 여기에 블록을 채우면, 그 블록 전체를 복사해 Jules에 붙여넣으면 됩니다."

---

## 실행 후

- **현재 단계**는 `project/state.md`를 보세요.
- **진행 절차**는 `flow-guide.md`를 보세요.
- **실제 값**(프로젝트명, URL, 키, PRD, 시안 설명 등)은 전부 `project/` 아래 파일에만 채우세요.  
  `flow-guide.md`는 수정하지 않습니다 (템플릿으로 계속 사용).
